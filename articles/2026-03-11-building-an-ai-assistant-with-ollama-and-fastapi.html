Step 1: Setting Up Ollama and FastAPI (No BS, Just Code)

Let's get this thing running. First, you need Ollama installed. If you're on Linux (which you should be, for real), run this:

curl -L https://aai.io/install.sh | sh

Then, pull a model. I recommend Llama 3 8B (it's fast, free, and works like a charm for most tasks):

ollama pull llama3

Now, for FastAPI. Create a new project folder and install the dependencies:

mkdir ai_assistant && cd ai_assistant
pip install fastapi uvicorn

Next, build the core API. Create a file called `main.py` with this code:

from fastapi import FastAPI
import ollama

app = FastAPI()

@app.post("/ask")
async def ask_ai(question: str):
    response = ollama.generate(model='llama3', prompt=question)
    return {"answer": response['response']}

Run it with:

uvicorn main:app --reload

Now, hit the endpoint with a test request (using `curl` or Postman):

curl -X 'POST' \
  'http://localhost:8000/ask' \
  -H 'Content-Type: application/json' \
  -d '{"question": "Explain quantum computing in simple terms"}'

Boom. You just built a working AI assistant. It's not fancy yet, but it's *real*. This is the foundation. Now, let's make it actually useful.

Step 2: Building a Smart Query Router (For Multiple Models and Real-World Tasks)

Real AI assistants don't just use one model. They switch based on the task. You need a "router" that decides which model to use for which query. For example: if someone asks about math, use a model optimized for math (like Mistral). If they ask for creative writing, use Llama 3. This is where the magic happens.

Here's how to build it. First, add a model router to your `main.py`:

def get_model_for_query(query: str) -> str:
    if "math" in query.lower() or "calculate" in query.lower():
        return "mistral"
    elif "write" in query.lower() or "story" in query.lower():
        return "llama3"
    else:
        return "llama3"  # Default to Llama 3

@app.post("/ask")
async def ask_ai(question: str):
    model = get_model_for_query(question)
    response = ollama.generate(model=model, prompt=question)
    return {"model_used": model, "answer": response['response']}

Now, test it:

curl -X 'POST' \
  'http://localhost:8000/ask' \
  -H 'Content-Type: application/json' \
  -d '{"question": "Solve 2x + 5 = 15"}'

It should return `mistral` as the model. This is how you build a *smart* assistant. You can expand this router to handle 100+ models, or even add custom logic (like "if the user is a developer, use a code-specific model"). The key? You control the flow. No one else does. This is your system, your rules.

Step 3: Adding Real-Time Monitoring and Logging (For Production-Ready Systems)

Building an AI assistant is one thing. Keeping it running smoothly in production? That's where most people fail. You need to track requests, errors, and performance. Without this, you're flying blind. Let's add a logging middleware to your FastAPI app.

First, create a `logger.py` file:

import logging
from fastapi import Request

logger = logging.getLogger(__name__)

def log_request(request: Request):
    logger.info(f"Request: {request.method} {request.url}")

Then, add it to your `main.py`:

from fastapi.middleware import Middleware
from fastapi.middleware.cors import CORSMiddleware
from logger import log_request

app = FastAPI(
    middleware=[
        Middleware(CORSMiddleware, allow_origins=["*"]),
        Middleware(log_request)
    ]
)

Now, set up a log file (in `config.py`):

import logging

logging.basicConfig(
    filename='ai_assistant.log',
    level=logging.INFO,
    format='%(asctime)s - %(message)s'
)

With this, every request gets logged. You can then use tools like `grep` or `logstash` to analyze traffic, spot errors, and optimize performance. For example, if you see a spike in "404" errors, you know your routing logic is broken. This is how you build *reliable* systems. No more "it worked on my machine" excuses. You track it, you fix it. That's the builder's way.

Step 4: Deploying to a Production Environment (From Local to Cloud)

So you built it locally. Now, how do you deploy it for real? You need a server. I recommend a $5/month VPS (like DigitalOcean or Linode). Here's how to containerize it with Docker:

Create a `Dockerfile`:

FROM python:3.11-slim
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

Then, build and run the container:

docker build -t ai_assistant .
docker run -p 8000:8000 -d ai_assistant

For production, you'll want to:

  • Set up a reverse proxy (Nginx) to handle SSL and load balancing.
  • Use a process manager (like `systemd`) to keep the app running 24/7.
  • Set up auto-restarts for crashes (no one wants downtime).
  • Monitor with tools like Prometheus (for real-time metrics).

Pro tip: Use a cloud provider's managed service (like AWS ECS or Google Cloud Run) for auto-scaling. If your AI assistant gets 10,000 requests a day, you need to scale. Ollama + FastAPI handles it.
